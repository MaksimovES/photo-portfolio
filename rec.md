## 1. GLOBAL — Весь сайт

### 1.1 Кастомный курсор
**Приоритет: высокий.**
На всех топовых editorial-сайтах (KOTA, Bureau, Superhero Cheesecake) — кастомный курсор.
- Маленький круг `12px` gold цвет, следует за мышью с `useSpring` (небольшой lag = органика)
- При наведении на фото — увеличивается до `80px`, внутри текст `"СМОТРЕТЬ"` (буквы по кругу через CSS `letter-spacing` + `rotate` или SVG `textPath`)
- При наведении на кнопки/ссылки — уменьшается до `6px` и меняет форму на крест
- При клике — мгновенный scale `0.7` с возвратом
- Компонент `<Cursor />` в `Layout.tsx`, скрыть нативный курсор через `cursor: none` только на десктопе (`@media (pointer: fine)`)

### 1.2 Плавный скролл (Lenis)
**Приоритет: высокий.**
Сейчас стандартный браузерный скролл. Библиотека **Lenis** (от Studio Freight) добавляет кинетический скролл с инерцией — именно это делает Awwwards-сайты "живыми".
```bash
npm install lenis
```
Инициализировать в `Layout.tsx` через `useEffect`, интегрировать с Framer Motion `useScroll` через `lenis.on('scroll', ...)`.

### 1.3 Page Transition — цветовая шторка
**Приоритет: средний.**
Текущий переход между страницами — простой fade/slide opacity. Топовые сайты используют цветную шторку.
Предложение: при переходе `inset(0 0 0% 0)` → `inset(0 0 100% 0)` на тёмном `#1A1008` оверлее, затем новая страница появляется снизу. Длительность: `0.6s`. Это уже частично реализовано в MenuOverlay — применить ту же механику к Layout transitions.

### 1.4 Noise / Grain на фоне — сделать динамическим
**Что сейчас:** статичный SVG-шум через `body::after`, анимированный только размером `200px`.
**Улучшение:** добавить `animation: grainMove 0.5s steps(1) infinite` который смещает `background-position` случайным образом — получится живая кинозернистость как у Cossette/Resn. Это 3 строки CSS.

### 1.5 Scroll Progress Bar
Тонкая линия `1px` gold цвета вдоль левого края экрана (или сверху) показывающая прогресс скролла страницы. `scaleY` через `useScroll` + `useSpring`. На жанровых страницах с галереями — очень полезна.

### 1.6 "Выбрать язык" / локализация
Незначительно, но для международного портфолио — добавить RU / EN переключатель в Header, одна кнопка с плавной сменой.

## 3. HERO

### 3.1 Параллакс на фото при скролле
**Приоритет: высокий.**
Фото в Hero должно слегка съезжать вниз медленнее страницы при скролле — классический parallax.
```tsx
const { scrollY } = useScroll()
const y = useTransform(scrollY, [0, 500], [0, 80])
// применить к motion.div обёртке фото
```
Это один из самых мощных визуальных приёмов и на сайте его нет.

### 3.2 Горизонтальная полоса с датой / координатами
Добавить строку метаданных под или над заголовком — как у многих editorial-фотографов:
```
59°56′N 30°18′E  ·  Санкт-Петербург  ·  2024
```
`text-[10px] tracking-[0.3em] text-warm-muted/40` — очень тонко и атмосферно.

### 3.3 Scroll Indicator
Внизу Hero — анимированная стрелка вниз или текст `SCROLL` с линией которая "стекает" вниз. Исчезает когда пользователь начинает скроллить.
```tsx
const opacity = useTransform(scrollY, [0, 100], [1, 0])
```

### 3.4 Hover-эффект на фото
При наведении на фотографию в Hero — лёгкий `scale(1.03)` с `transition: transform 0.8s` и появление тонкого золотого оверлея `bg-gold/5`. Ощущение интерактивности.

### 3.5 Счётчики — добавить разделитель и единицы
Сейчас: `127 / историй снято`. Предложение: добавить вертикальную линию `w-px h-8 bg-gold/20` между счётчиками. И сделать их кликабельными (ведут к соответствующим секциям).

## 4. HORIZONTALSCROLL ("Примеры работ")

### 4.1 Сделать настоящий горизонтальный скролл
**Это самое сильное что можно добавить на главную.**
Переименовать секцию и реализовать настоящий горизонтальный скролл при вертикальном scroll пользователя — `pin` секции через scroll и двигать содержимое по X. Реализуется через:
- `useScroll` + `useTransform` scrollY → translateX
- Или через `ScrollTrigger` (GSAP) / нативный `sticky` + JS

Карточки: реальные фото `3:4`, при hover — золотой оверлей с жанром, клик → страница жанра. Количество: 8–12 фото.

### 4.2 Крупный текст-бегущая строка (Marquee) между секциями
Между HorizontalScroll и Cases добавить бегущую строку:
`СВАДЬБА · СТУДИЯ · ТЕАТР · СЕМЬЯ · РЕПОРТАЖ · STREET · ` повторяется бесконечно.
`font-display`, крупный шрифт `clamp(3rem, 8vw, 6rem)`, очень медленная скорость, цвет `text-cream-300/20`.
Это один из самых узнаваемых приёмов Awwwards-сайтов.

## 5. CASES ("Направления")

### 5.1 Превью фото при открытии аккордеона
Правая колонка сейчас пустая. При открытии жанра — показывать фото из этого жанра:
```tsx
// при isOpen показывать первое фото из GENRE_PHOTOS[genre.slug]
<AnimatePresence mode="wait">
  {openGenre && (
    <motion.img
      key={openGenre}
      src={GENRE_PHOTOS[openGenre][0]}
      initial={{ opacity: 0, scale: 1.05 }}
      animate={{ opacity: 1, scale: 1 }}
      exit={{ opacity: 0 }}
      transition={{ duration: 0.5 }}
    />
  )}
</AnimatePresence>
```

### 5.2 Hover — показывать превью рядом с курсором
При наведении на название жанра — небольшое `240×320px` превью-фото появляется рядом с курсором и следует за ним. Называется "magnetic image follow". Реализуется через `onMouseMove` + `useSpring`. Один из самых залипательных эффектов на Awwwards.

### 5.3 Счётчик фото в жанре
В строке жанра добавить количество фото: `— 24 снимка`. `font-body text-2xs text-warm-muted/50`.

### 6.1 Заменить кнопочный слайдер на drag-to-scroll

### 6.2 Добавить фото клиентов (аватары)
Каждая карточка — добавить маленький круглый аватар `40px` рядом с именем. Если нет реального фото — нейтральные плейсхолдеры или первая буква имени на золотом фоне.

### 6.3 Рейтинг / звёзды
Добавить 5 золотых звёзд или `★★★★★` под именем — классический элемент доверия.

### 6.4 Индикаторы слайдов
Сейчас нет визуального счётчика какой отзыв из скольки. Добавить под слайдером:
`— 01 / 04 —` с тонкой прогресс-линией которая заполняется.

### 6.5 Автоплей с паузой на hover
Карусель должна автоматически переключаться каждые `5–6s`, пауза при наведении мыши.

## 14. МИКРОВЗАИМОДЕЙСТВИЯ

### 14.1 Magnetic Buttons
Кнопки при приближении курсора — притягиваются к нему (magnetic effect). Реализуется через `onMouseMove` + `useSpring` на `x` и `y` смещение кнопки.
Особенно эффектно для CTA кнопок ("Направления съемок", "Написать фотографу").

### 14.2 Hover на фото в сетке — scale + оверлей
В GenrePhotoGrid при hover на фото:
- `scale(1.04)` плавно за `0.6s`
- Тёмный оверлей `bg-warm-dark/20`
- Появляется номер фото `07` крупным шрифтом в центре
- Курсор меняется на кастомный (см. п. 1.1)

### 14.3 Кнопки — shimmer-эффект при hover
На кнопке "Направления съемок" в Hero — при hover добавить shimmer sweep слева направо (уже есть анимация в CSS). Применить к основным CTA-кнопкам.

### 14.4 Числа — scramble-эффект
При появлении счётчиков (127, 8, 4) — перед финальным числом показывать быстро мелькающие случайные цифры (scramble). Библиотека `react-scramble` или 20 строк кода. Выглядит очень технично.

## 15. ПРОИЗВОДИТЕЛЬНОСТЬ И ТЕХДОЛГ

### 15.1 Реальные фото вместо picsum
Сейчас Hero фото: `https://picsum.photos/seed/portfolio/480/640`. В продакшне — заменить на реальное фото через локальный импорт или CDN.

### 15.2 WebP формат
Все фото должны быть в `WebP` — в 2–3 раза меньше размер при том же качестве.

### 15.3 Blur placeholder для фото
Добавить `blurDataURL` (base64 миниатюра `4×4px`) пока грузится полная версия. Убирает белый мигающий прямоугольник даже с shimmer.

### 15.4 Виртуализация больших галерей
Если фото в жанре > 30 — использовать `react-virtual` или `@tanstack/react-virtual` для рендера только видимых фото. Сейчас все рендерятся сразу.

### 15.5 OG meta-теги
Добавить `<meta og:image>` `<meta og:title>` для каждой страницы — красивый превью при шаринге ссылки.

### 15.6 Favicon
Добавить favicon в стиле монограммы `KM` — сейчас скорее всего стандартный Vite.
