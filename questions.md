# Открытые вопросы перед началом разработки

## 1. Структура и навигация
- [x] Какие страницы будут на сайте?
  > **Главная** (фотки, карусели, краткая инфа, FAQ-блок) + **отдельные страницы по жанрам съёмок** (переход из меню) + **Обо мне**
- [x] Одностраничник (SPA с секциями) или многостраничник с роутингом?
  > **Многостраничник с роутингом** — главная + страницы жанров + обо мне
- [x] Меню: бургер на мобилке, горизонтальное на десктопе? Фиксированное сверху?
  > **Модальное окно** (fullscreen overlay) с красивой анимацией. Содержит: ссылки на жанры съёмок + кнопка «Обо мне». Одинаковый формат на мобилке и десктопе.

## 2. Дизайн и стиль
- [x] Цветовая схема: тёмная / светлая / смешанная?
  > **Тёплые тона**, дорогие удачные сочетания. Фон должен обыгрывать фотографии — подберём фотки на главную под цветовую схему, чтобы всё смотрелось как единое целое.
- [x] Шрифты: минималистичные sans-serif или что-то более авторское?
  > **Два типа шрифтов:** каллиграфический (акцентный, для заголовков/декора) + чёткий дорогой шрифт (для основного текста/UI). Премиальная типографика.
- [x] Язык сайта: русский, английский, оба?
  > **Русский**
- [x] Общий вайб: строгий/минималистичный, тёплый/уютный, яркий/креативный?
  > **Премиальный, тёплый, дорогой.** Всё должно выглядеть шикарно — «очень дорого делаем».

## 3. Главная страница
- [x] Что в hero-секции?
  > **Краткое представление + фото самого фотографа** с красивой анимацией появления
- [x] Intro-анимация — какая именно?
  > **Intro-анимации нет.** Вместо этого — **красивый preloader** в стиле Awwwards-сайтов (плавная загрузка, а не моргание страницы). После preloader — плавное появление контента.
- [x] Какие секции на главной и в каком порядке?
  > 1. **Hero** — представление + фото фотографа + анимация появления
  > 2. **Горизонтальный скролл фотографий** — срабатывает автоматом при вертикальном скролле
  > 3. **Кейсы / достижения** — что крутого сделал фотограф, описание кейсов с фотками
  > 4. **Отзывы**
  > 5. **FAQ**
  > ⚠️ Это не финальный список — будет дополняться по ходу, когда будет видно результат

## 6. Отзывы
- [x] Статичные (добавляешь сам) или клиенты оставляют через сайт?
  > **Клиенты оставляют через сайт**, но фотограф модерирует: может одобрить, изменить, добавить фотки к отзыву.
- [x] Формат и логика отзывов?
  > **Отзыв — это мини-кейс.** Клик по отзыву → переход на страницу съёмки этого человека, где виден сам отзыв + фотографии со съёмки. Связка: отзыв ↔ съёмка ↔ фотографии.
- [ ] Дизайн: карусель, карточки, список?
- [x] Как фотограф управляет отзывами?
  > **Админ-панель на сайте**, mobile-first — всё можно делать с телефона фотографа. Функции: одобрить/отклонить/редактировать отзыв, прикрепить фотки к отзыву (автоматически красиво располагаются), опубликовать одной кнопкой — сразу появляется на сайте.

## 8. Анимации
- [x] Где именно нужны анимации?
  > **Scroll-driven анимации по всему сайту.** Контент адаптивно двигается от скролла: вертикальный скролл переходит в горизонтальный скролл фотографий, затем появление следующего блока с анимациями. Параллакс, плавные появления — премиальный уровень.
- [x] Three.js — точно нужна или хватит GSAP / Framer Motion?
  > **Без Three.js.** Используем GSAP (ScrollTrigger) + Framer Motion.
- [ ] Intro-анимация: что конкретно? Логотип? Текст? 3D-объект?

## 10. Технические решения
- [x] Роутинг: React Router?
  > **Да, React Router.** Стандарт для React + Vite, покрывает все наши страницы (главная, жанры, обо мне, админка, страница съёмки).
- [x] Стили: CSS Modules, Tailwind, Styled Components, SCSS?
  > **Tailwind CSS.** Опционально — компоненты из 21st.dev точечно по необходимости.
- [x] Нужен ли бэкенд?
  > **Да.** Админ-панель + отзывы + загрузка фото требуют полноценного бэкенда.
  >
  > **Стек бэкенда:**
  > - **Node.js + Express** — один язык с фронтом, просто деплоить на VPS
  > - **PostgreSQL** — реляционная БД (отзыв → съёмка → фотографии)
  > - **Prisma** — ORM, миграции, типобезопасность
  > - **Sharp** — оптимизация и превью фото при загрузке
  > - **Хранение фото** — локально на VPS
  > - **JWT** — авторизация админа
  >
  > **Безопасность (необходимый минимум):**
  > - HTTPS (Let's Encrypt)
  > - Bcrypt — хэширование пароля админа
  > - JWT в HTTP-only cookie
  > - Rate limiting на логин
  > - Валидация загружаемых файлов (тип, размер)
- [x] Хостинг: Vercel, Netlify, GitHub Pages?
  > **Свой VPS + купленный домен**
- [x] SEO: нужен ли SSR/SSG? (Тогда Next.js вместо Vite)
  > **Нет, Vite.** Трафик идёт по прямым ссылкам (Instagram) или конкретным запросам, SEO не приоритет.
