## Фаза 4 — Бэкенд

### 4.1 Инициализация бэкенда
- [ ] Node.js + Express + TypeScript
- [ ] Структура папок: `routes/`, `controllers/`, `middleware/`, `services/`, `prisma/`
- [ ] Prisma + PostgreSQL: подключение, начальная схема
- [ ] Базовые middleware: CORS, Helmet, rate limiting, body parser
- [ ] Переменные окружения (.env + .env.example)

### 4.2 Схема БД и модели
- [ ] Модель `Admin` — логин/пароль админа
- [ ] Модель `Genre` — жанры съёмок (название, slug, описание, порядок)
- [ ] Модель `Shoot` — съёмка (связь с жанром, название, описание, дата)
- [ ] Модель `Photo` — фотография (связь с жанром и/или съёмкой, путь, alt, порядок)
- [ ] Модель `Review` — отзыв (связь со съёмкой, имя клиента, текст, статус модерации)
- [ ] Модель `FAQ` — вопрос-ответ (текст, порядок)
- [ ] Миграции Prisma

### 4.3 Авторизация админа
- [ ] Эндпоинт `POST /api/auth/login` — вход (bcrypt + JWT)
- [ ] Middleware `requireAuth` — проверка JWT из HTTP-only cookie
- [ ] Эндпоинт `POST /api/auth/logout`
- [ ] Безопасность: rate limiting на логин, secure cookies

### 4.4 API: Жанры и фотографии
- [ ] CRUD жанров (`/api/genres`)
- [ ] Загрузка фотографий (`/api/photos/upload`) — multer + Sharp (ресайз, превью)
- [ ] CRUD фотографий (`/api/photos`) — привязка к жанру/съёмке
- [ ] Валидация файлов: тип, размер, содержимое

### 4.5 API: Съёмки и отзывы
- [ ] CRUD съёмок (`/api/shoots`)
- [ ] Публичный эндпоинт: оставить отзыв (`POST /api/reviews`)
- [ ] Админ эндпоинты: модерация отзывов (одобрить / отклонить / редактировать)
- [ ] Привязка фотографий к отзыву/съёмке
- [ ] Публичный эндпоинт: получить одобренные отзывы

### 4.6 API: FAQ
- [ ] CRUD FAQ (`/api/faq`) — админ
- [ ] Публичный эндпоинт: получить FAQ
